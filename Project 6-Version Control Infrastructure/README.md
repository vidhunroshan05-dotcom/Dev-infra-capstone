# Project 6 - Version Control Infrastructure

## Goal

Track Terraform infrastructure changes using Git and Pull Requests

---

## Description

This project demonstrates how infrastructure code is managed safely using Git.

A Terraform change was made in a separate feature branch and merged into the main branch using a Pull Request, following standard DevOps practices.

---

## Changes Made

Updated the EC2 instance type from **t2.micro** to **t2.small**.

---

## Workflow Followed

* Created a feature branch
* Modified the Terraform configuration
* Committed the change to the feature branch
* Pushed the branch to GitHub
* Raised a Pull Request
* Merged the Pull Request into main

## Conclusion

This project highlights the importance of version control in infrastructure management. By using feature branches and Pull Requests, infrastructure changes can be reviewed, tracked, and safely merged. This approach reduces risk and improves collaboration in real-world DevOps environments.
